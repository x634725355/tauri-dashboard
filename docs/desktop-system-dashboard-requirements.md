# 桌面系统仪表盘 — 需求与设计输入（已确认）

> 状态：**理解锁定已确认**（2026-04-20）  
> 范围：Windows 与 macOS 双平台桌面应用

本文档归档自需求澄清与决策记录，用于后续实现与评审对照。

---

## 1. 理解摘要

1. **产品形态**：常驻 **主窗口仪表盘**；**窗口大小可调节**；数据整体刷新基线约 **1 次/秒**。
2. **展示指标**：**CPU 温度**；**主 GPU 温度**（仅展示一块「主 GPU」）；**主显示器刷新率（Hz）**；**本地固定磁盘与可移动磁盘的容量**；**系统总网络上下行吞吐**；**内存占比**。
3. **控制项**：**系统主音量**；**屏幕亮度**，并 **尽量覆盖外接显示器/多屏**（接受更多平台差异、不可用分支与说明文案）。
4. **平台**：**Windows** 与 **macOS** 均需兼容。
5. **受众与容错**：主要 **自用**；部分硬件/驱动下指标 **读不到则显示不可用（N/A）** 可接受。
6. **启动**：需要 **随用户登录自动启动**。
7. **明确边界（由澄清阶段推断，已获确认）**：不做游戏/全屏 **实时渲染帧率** 或注入式采集；「帧率」指 **主显示器刷新率**；网速为 **系统总吞吐** 口径（非仅默认路由单链路）；不要求应用内 **切换音频输出设备**；磁盘统计 **不含网络映射盘**（相对「尽量全」更克制）。

---

## 2. 非功能需求（默认基线）


| 维度    | 约定                                                               |
| ----- | ---------------------------------------------------------------- |
| 性能    | 以 **~1Hz** 刷新为主；长期运行 **CPU 占用尽量低**；采集与 UI 解耦，避免阻塞渲染。             |
| 规模    | 自用 **单机**；无多租户/服务端聚合需求。                                          |
| 安全与隐私 | 默认 **不上传** 监控数据至云端；若后续增加自动更新，需单独定义「更新检查」出网策略。                    |
| 可靠性   | 权限不足、驱动缺失、接口不可用时 **降级为 N/A** 并附 **简短原因**；外接屏亮度、部分温度源在平台上可能常态不可用。 |
| 维护    | 个人维护为主；接受 **平台分支** 与 **能力探测**（feature detection）。                |


---

## 3. 假设（实现前仍可在 ADR 中细化）

1. **「主 GPU」** 与系统「首选/主渲染 GPU」的认定，以 **各平台设备枚举 + 合理启发式** 为准；不承诺覆盖所有厂商图形切换策略的极端情况。
2. **CPU 温度** 以 **单一代表性读数** 展示；若底层暴露多传感器，再在实现阶段定义聚合策略（并在文档中注明）。
3. **内存占比** 按 **各平台常见口径**（如物理已用/总量一类）实现；**允许** 与部分第三方工具存在小幅显示差异。
4. **第一版** 不强制 **系统托盘/菜单栏常驻**（主窗口为澄清阶段选定形态）；若自启动后需「最小化到托盘」，作为后续增量评估。

## 4. 决策日志


| 决策项      | 选定方案                     | 备注               |
| -------- | ------------------------ | ---------------- |
| 产品形态     | **A** 主窗口仪表盘             | 非托盘优先形态          |
| 窗口       | **可调节大小**                | UI 按响应式布局设计      |
| 刷新节奏     | **A** 约 **1 次/秒**        | 全局基线             |
| 分发与受众    | **A** 主要自用               | 可读性缺失可接受         |
| 「帧率」含义   | **A** 主显示器 **刷新率（Hz）**   | 非游戏帧率、非本应用渲染 FPS |
| 亮度覆盖     | **B** 尽量多屏/外接            | 复杂度高、不可用分支多      |
| 音量       | **A** **系统主音量**          | 不含应用内切换输出设备      |
| 网速口径     | **A** **系统总吞吐**（多网卡合并观感） | 非仅默认路由接口         |
| 磁盘范围     | **B** 本地固定盘 **+** 可移动盘   | 不含网络映射盘          |
| 自启动      | **B** 登录后自动启动            | 平台登录项/启动项差异需单独设计 |
| 多 GPU 温度 | **A** 仅 **主 GPU** 一行     | 另一块不展示           |

## 5. 开放问题（留待架构/实现阶段）

以下项 **不否定当前需求**，但需在技术方案中逐项落地：

1. **温度读数来源**：各平台 API、系统扩展、或依赖库的选型；以及可能触发的 **权限/隐私弹窗** 清单（尤其 macOS）。
2. **多显示器亮度**：macOS 上外接屏常 **不可调** 时的 UI（灰显、按显示器分组、重试策略）与文案。
3. **自启动体验**：macOS **登录项/后台项** 与 Windows **启动项** 的配置路径及用户引导。
4. **主 GPU 判定**：与「主显示器」关联时的边界情况（无独显、热插拔、eGPU 等）。
